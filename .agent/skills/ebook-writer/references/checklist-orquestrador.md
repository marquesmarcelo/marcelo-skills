# Checklist do Orquestrador — Revisão de Módulos

Use esta lista ao revisar cada rascunho gerado pelos agentes subsidiários.

---

## ✅ Checklist Estrutural

### Metadados e Cabeçalho
- [ ] Título do módulo presente e claro
- [ ] Nome do curso e numeração do módulo (ex: "Módulo 03 de 08") presentes
- [ ] Bloco "📌 Neste módulo você vai:" com 3 bullets motivadores no início

### Seções Obrigatórias
- [ ] `## 🎯 Introdução` — presente e com 2–3 parágrafos contextualizadores
- [ ] `## 🏁 Objetivos de Aprendizagem` — presente com 5 objetivos usando verbos de ação
- [ ] `## 📚 Conteúdo` — presente com pelo menos 3 subseções
- [ ] `## 🗺️ Resumo` — presente com parágrafo de fechamento E tabela de conceitos
- [ ] `## 🚀 Próximos Passos` — presente com lista de ações E teaser do próximo módulo
- [ ] `## 🖼️ Imagens Sugeridas` — presente com 3–5 imagens

---

## ✅ Checklist de Extensão

- [ ] Contagem de palavras estimada: entre 4.000 e 7.500 palavras
- [ ] Número de subseções no Conteúdo: pelo menos 3
- [ ] Cada subseção tem pelo menos 3 parágrafos

**Como estimar páginas:**
- 1 página ≈ 400–500 palavras (considerando elementos visuais e espaçamento)
- Meta: 10–15 páginas = 4.000–7.500 palavras
- Se curto: adicionar exemplos práticos, casos de uso, ou mais detalhes técnicos
- Se longo: identificar partes redundantes para condensar

---

## ✅ Checklist de Elementos de Engajamento

| Elemento | Mínimo | Encontrado? |
|----------|--------|-------------|
| 💡 Dica | 3 | [ ] |
| 🔍 Saiba Mais | 2 | [ ] |
| ⚠️ Atenção | 2 | [ ] |
| 🚀 Desafio | 1 | [ ] |
| 💬 Curiosidade | 1 | [ ] |

- [ ] Elementos distribuídos ao longo do texto (não concentrados numa seção)
- [ ] Cada `🔍 Saiba Mais` contém pelo menos 2 recursos externos com URLs ou descrições

---

## ✅ Checklist de Imagens

- [ ] Entre 3 e 5 imagens no arquivo
- [ ] Cada imagem tem: nome, posição no texto, tipo, prompt NanaBanana Pro em inglês, descrição em português
- [ ] Marcação `[IMAGEM: nome]` inserida no local correto dentro do conteúdo
- [ ] Prompts têm no mínimo 50 palavras
- [ ] Prompts especificam: estilo visual, elementos, cores, mood, formato/proporção

---

## ✅ Checklist de Tom e Linguagem

- [ ] Usa "você" consistentemente (não "o aluno" ou "os estudantes")
- [ ] Parágrafos curtos (3–5 linhas no máximo)
- [ ] Pelo menos 2 analogias ou metáforas com o mundo real
- [ ] Termos técnicos em **negrito** na primeira ocorrência + explicação
- [ ] Linguagem descontraída mas tecnicamente precisa
- [ ] Sem blocos de texto muito densos (mais de 8 linhas sem quebra)

---

## ✅ Checklist Técnico

- [ ] Conteúdo tecnicamente correto e atualizado
- [ ] Exemplos são práticos e relevantes para profissionais de TI
- [ ] Nível de profundidade adequado ao público-alvo definido
- [ ] Progressão lógica: cada subseção parte do que foi visto antes
- [ ] Objetivos de aprendizagem são alcançados pelo conteúdo apresentado

---

## ✅ Checklist de Continuidade (quando módulo não é o último)

- [ ] Próximos Passos menciona o módulo seguinte pelo nome/título correto
- [ ] Teaser do próximo módulo é motivador e preciso
- [ ] Há pelo menos uma referência a conteúdo que será visto em módulos futuros (quando pertinente)

---

## 🔧 Ações Corretivas por Problema

| Problema | Ação |
|----------|------|
| Texto muito curto (< 4.000 palavras) | Expandir subseções com mais exemplos práticos e casos de uso |
| Texto muito longo (> 7.500 palavras) | Identificar redundâncias; mover conteúdo avançado para "Saiba Mais" |
| Faltando elemento de engajamento | Inserir no ponto mais relevante do texto |
| Tom formal demais | Reescrever parágrafos introdutórios de cada subseção com analogias |
| Objetivos fracos (sem verbos de ação) | Reescrever usando taxonomia de Bloom: identificar, configurar, implementar, comparar, analisar... |
| Prompt de imagem vago | Complementar com: estilo visual, paleta de cores, formato e lista de elementos |
| Conteúdo tecnicamente impreciso | Reescrever a passagem com informação correta e marcar para revisão humana |
